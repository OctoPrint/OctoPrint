--- layout: post author: foosel title: 'New release: 1.2.7' date: 2015-10-20 12:50:03 +0000 originalUrl: https://plus.google.com/+OctoprintOrg/posts/PnuXJbjrWjW --- Mostly fixes of minor bugs and annoyances but also a couple of improvements:

  * The Plugin Manager now has a configuration dialog that allows fine tuning some settings for more uncommon types of setups.
  * Timelapse processing has been improved. A capture queue now ensures that individual snapshots can never be taken out of order and notifications will be shown when capturing of the post roll takes place (incl. an approximation of how long it will take).
  * Downloaded GCODE and STL files will now have a Content-Type text/plain respectively application/sla set, allowing desktop applications (editors, previewers, ...) to be properly associated.
  * Configuration files are now saved in a more error resilient way, hopefully helping prevent corruption in case of file system hiccups during saving.

Find the full changelog here: https://github.com/foosel/OctoPrint/releases/tag/1.2.7

If you don't get an "Update Now" button with your update notification, read this: https://github.com/foosel/OctoPrint/wiki/Plugin:-Software-Update#making-octoprint-updateable-on-existing-installations or - even more specifically - this: https://github.com/foosel/OctoPrint/wiki/Plugin:-Software-Update#octoprint--125
